# 核心业务流程总览

## 1. 商户全生命周期流程

```
┌──────────────────────────────────────────────────────────────────────┐
│                        商户全生命周期                                 │
│                                                                      │
│  ① 注册        ② KYB认证      ③ 入驻申请      ④ 技术集成     ⑤ 运营  │
│  ─────────  → ──────────  → ──────────  → ──────────  → ────────── │
│  邮箱注册     提交公司资质    提交业务信息    API对接        交易监控   │
│  邮箱验证     法人信息       上传合规文件    签名调试       订单管理   │
│  首次登录     合规文件       等待审核       Webhook配置    数据导出   │
│              等待审核       审核通过       沙箱测试       异常处理   │
│              审核通过                     上线生产                   │
└──────────────────────────────────────────────────────────────────────┘
```

## 2. 核心业务流程

### 2.1 商户注册与认证流程

```
用户访问平台
    │
    ├── [新用户] → 注册页面
    │       ↓
    │   填写注册信息（邮箱、密码、公司名称）
    │       ↓
    │   发送验证邮件
    │       ↓
    │   点击验证链接 → 注册完成 → 登录页面
    │
    └── [已注册用户] → 登录页面
            ↓
        输入邮箱 + 密码
            ↓
        [登录成功]
            ↓
        检查 KYB 状态
            ├── [未完成] → KYB 引导页
            │       ↓
            │   填写公司信息 → 法人信息 → 上传文件 → 提交
            │       ↓
            │   等待审核 → [通过] → 进入平台主页
            │            → [拒绝] → 修改重提
            │
            └── [已完成] → 进入平台主页
```

### 2.2 商户入驻申请流程

```
进入入驻申请页面
    │
    ├── [有草稿] → 恢复草稿，继续填写
    │
    └── [无草稿] → 开始新申请
            │
            ├── [生产环境 - 4步]
            │   步骤1: 公司信息 → [自动保存草稿]
            │   步骤2: 业务信息 → [自动保存草稿]
            │   步骤3: 文件上传 → [自动保存草稿]
            │   步骤4: 确认并提交
            │
            └── [沙箱环境 - 3步]
                步骤1: 公司信息 → [自动保存草稿]
                步骤2: 业务信息 → [自动保存草稿]
                步骤3: 确认并提交

            ↓ [提交]
        申请进展追踪
            ↓
        已提交 → 审核中 → 已通过 / 已拒绝
            ↓
        [状态变更] → 邮件通知 + 平台内通知
```

### 2.3 技术集成流程（OpenAPI 模式）

```
技术人员登录平台
    ↓
① 获取凭证
    开发者控制台 → 复制 App ID + 公钥
    ↓
② 理解 API
    API 文档 → 浏览端点 → 复制 AI 提示词 → AI 生成代码
    ↓
③ 实现签名
    签名工具 → 在线测试签名逻辑 → 验证签名正确性
    ↓
④ 沙箱测试
    切换到沙箱环境 → 使用测试卡号 → 在线试用 API
    ↓
⑤ 配置 Webhook
    Webhook 管理 → 创建端点 → 选择事件 → 测试推送
    ↓
⑥ 配置域名白名单（WebSDK 模式需要）
    域名白名单 → 添加业务域名
    ↓
⑦ 上线生产
    切换到生产环境 → 更新端点地址 → 正式上线
```

### 2.4 技术集成流程（AI/MCP 模式）

```
技术人员登录平台
    ↓
① MCP 配置中心 → 复制 MCP 配置 JSON
    ↓
② 打开 AI 编程助手（Cursor / Windsurf）→ 粘贴配置 → 重启
    ↓
③ 自然语言调用
    "获取 100 USD 到 USDT 的报价"
    → AI 自动: SSE 连接 → 调用 MCP 工具 → 自动签名 → 返回结果
    ↓
④ AI 生成完整集成代码
    "帮我用 TypeScript 实现完整的支付流程"
    → AI 参考快速开始指南 → 生成代码 → 包含签名逻辑
    ↓
⑤ 沙箱测试 → 生产上线
```

### 2.5 支付订单全流程

```
终端用户发起支付
    ↓
商户系统调用 OSLPay API
    ↓
① 获取报价
    POST /api/v1/quote
    法币金额 + 目标加密货币 → 返回汇率和预估数量
    ↓
② 创建订单
    POST /api/v1/order
    报价ID + 用户信息 + 支付方式 → 返回订单ID + 支付链接
    ↓
③ 用户支付
    用户在支付页面完成法币支付（银行卡 / Google Pay / Apple Pay）
    ↓ [3DS 验证（如需要）]
    ↓
④ 支付处理
    CREATED → PROCESSING → SUCCESSED
    ↓
⑤ 加密货币发送
    OSLPay 向用户钱包地址发送加密货币
    SUCCESSED → COMPLETED
    ↓
⑥ Webhook 通知
    order_status_change → 推送到商户 Webhook 端点
    ↓
⑦ 商户确认
    商户系统查询订单状态（或接收 Webhook）→ 更新业务状态

    ※ 失败场景: PROCESSING → FAILED → Webhook 通知商户
```

### 2.6 Webhook 事件处理流程

```
系统事件触发（订单状态变更、KYC 审核等）
    ↓
Webhook 管理服务检查订阅配置
    ↓
构造 Webhook 消息（JSON Body + Headers）
    ↓
发送 HTTP POST 到商户端点
    ├── [2xx 响应] → 标记为 success → 记录日志
    └── [非 2xx / 超时] → 标记为 retry_pending
            ↓
        指数退避重试（1s → 2s → 4s → 8s → 16s）
            ├── [重试成功] → 标记为 success
            └── [5次全部失败] → 标记为 final_failed → 告警
```

## 3. 环境切换对业务流程的影响

| 流程 | 生产环境 | 沙箱环境 |
|------|----------|----------|
| 入驻申请 | 4 步（含文件上传） | 3 步（无文件上传） |
| KYB 认证 | 完整审核 | 简化/跳过 |
| API 端点 | `openapi.osl-pay.com` | `openapitest.osl-pay.com` |
| MCP 端点 | `mcp.osl-pay.com/sse` | `mcptest.osl-pay.com/sse` |
| 支付 | 真实扣款 | 测试卡号，不扣款 |
| KYC 信息 | 真实验证 | 不验证 |
| 数据 | 独立的生产数据 | 独立的测试数据 |
