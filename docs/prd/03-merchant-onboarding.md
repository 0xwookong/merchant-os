# 商户入驻申请

## 1. 功能概述

商户入驻申请模块支持商户在线提交业务入驻材料，系统根据当前环境（生产/沙箱）展示不同的申请流程。支持草稿自动保存、分多次填写、申请进度追踪等能力。

**适用角色**: 管理员、业务人员

## 2. 申请流程

### 2.1 生产环境 - 4 步流程

```
步骤1: 公司信息 → 步骤2: 业务信息 → 步骤3: 文件上传 → 步骤4: 确认提交
```

### 2.2 沙箱环境 - 3 步流程（无需上传文件）

```
步骤1: 公司信息 → 步骤2: 业务信息 → 步骤3: 确认提交
```

## 3. 表单字段设计

### 3.1 步骤 1：公司信息

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| 公司名称 | Text | ✅ | 商户公司全称 |
| 公司地址 | Text | ✅ | 详细地址 |
| 联系人姓名 | Text | ✅ | 主联系人 |
| 联系电话 | Phone | ✅ | 含国际区号 |
| 联系邮箱 | Email | ✅ | 业务联系邮箱 |

### 3.2 步骤 2：业务信息

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| 业务类型 | Select | ✅ | 下拉选择（电商、游戏、金融等） |
| 月交易量范围 | Select | ✅ | 预估月交易量区间 |
| 支持的法币 | Multi-Select | ✅ | USD, EUR, GBP, HKD 等 |
| 支持的加密货币 | Multi-Select | ✅ | BTC, ETH, USDT, USDC 等 |
| 业务描述 | Textarea | ✅ | 商户业务简述 |

### 3.3 步骤 3：文件上传（仅生产环境）

| 文件类型 | 格式 | 大小限制 | 必填 |
|----------|------|----------|:----:|
| 营业执照 | PDF/JPG/PNG | 10MB | ✅ |
| 法人身份证 | PDF/JPG/PNG | 10MB | ✅ |
| 银行对账单 | PDF/JPG/PNG | 10MB | ✅ |

### 3.4 步骤 4（生产）/ 步骤 3（沙箱）：确认提交

- 展示所有已填写信息的只读预览
- 展示已上传文件列表（生产环境）
- 提供「返回修改」和「提交申请」按钮
- 缺失必填项时「提交申请」按钮禁用，并标注缺失字段

## 4. 草稿管理

### 4.1 自动保存
- 每次点击「下一步」时自动保存当前进度为草稿
- 页面右上角显示当前状态标签：「草稿」

### 4.2 草稿恢复
- 用户离开页面后再次访问时，自动恢复草稿内容和当前步骤
- 草稿永久保存，直到用户提交或手动删除

### 4.3 演示重置
- 提供「重新提交（演示用）」按钮
- 将已提交的申请重置为草稿状态，方便多次演示

## 5. 申请状态流转

```
草稿(Draft)
    ↓ [用户提交]
已提交(Submitted)
    ↓ [系统确认收到]
审核中(Under Review)
    ├── [审核通过] → 已通过(Approved)
    └── [审核拒绝] → 已拒绝(Rejected)
```

### 状态说明

| 状态 | 标签颜色 | 说明 |
|------|----------|------|
| 草稿 | 灰色 | 用户正在填写，尚未提交 |
| 已提交 | 蓝色 | 用户已提交，等待处理 |
| 审核中 | 黄色/橙色 | 审核团队正在审核材料 |
| 已通过 | 绿色 | 审核通过，商户入驻成功 |
| 已拒绝 | 红色 | 审核未通过，需修改后重新提交 |

## 6. 申请进展追踪

提交后展示：
1. **状态时间线**: 提交申请 → 资料审核中 → 审核结果
2. **已填信息预览**: 公司信息、业务信息的只读展示
3. **文件列表**: 已上传文件的名称和状态（生产环境）
4. **状态变更通知**: 通过邮件和平台内通知告知业务人员

## 7. 业务规则

1. 必填项缺失时阻止提交，明确标注缺失字段
2. 文件格式限制：PDF、JPG、PNG
3. 单个文件大小限制：10MB
4. 沙箱环境简化流程，无需上传合规文件
5. 审核拒绝后支持修改并重新提交
6. 每个商户同一时间只能有一个有效的入驻申请

- [ ] 点击「下一步」自动保存草稿
- [ ] 离开页面后再次访问，恢复草稿内容和步骤
- [ ] 页面右上角正确展示申请状态标签
- [ ] 文件上传支持 PDF/JPG/PNG，限制 10MB
- [ ] 必填项缺失时阻止提交并标注缺失字段
- [ ] 提交后展示时间线和已填信息预览
- [ ] 状态变更时发送邮件和平台通知
- [ ] 「重新提交」按钮可将申请重置为草稿
