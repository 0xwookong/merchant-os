# 用户角色与权限体系

## 1. 角色定义

系统支持三种用户角色，每个角色拥有不同的功能访问范围：

### 1.1 管理员（Admin）
- **定位**: 商户的全权管理者
- **权限范围**: 全部功能
- **核心职责**:
  - 管理商户账户和系统配置
  - 邀请/移除团队成员，分配角色
  - 查看所有业务数据和审计日志
  - 提交和管理入驻申请

### 1.2 业务人员（Business）
- **定位**: 负责商户日常业务运营
- **权限范围**: 业务相关功能
- **核心职责**:
  - 提交和跟踪入驻申请
  - 查看交易数据和订单状态
  - 导出业务报表

### 1.3 技术人员（Technical）
- **定位**: 负责 API 集成和技术对接
- **权限范围**: 技术集成功能
- **核心职责**:
  - 浏览 API 文档，生成 AI 提示词和代码示例
  - 管理 API 凭证和域名白名单
  - 配置 Webhook 端点
  - 使用签名工具调试
  - 查看 API 请求日志

## 3. 权限控制规则

### 3.1 菜单可见性
- 系统根据用户角色动态过滤侧边栏菜单，仅显示有权限的菜单项
- 包含子菜单的一级菜单，如果所有子菜单均不可见，则该一级菜单也隐藏

### 3.2 页面访问控制
- 用户尝试访问无权限页面时，返回权限不足提示并重定向到授权页面
- 前端路由层和后端接口层均需做权限校验（双重保障）

### 3.3 审计日志
- 所有用户操作记录审计日志
- 日志内容：操作时间、操作人、操作类型、操作结果
- 审计日志不可篡改，用于合规审查

## 4. 验收标准

- [ ] 三种角色均可正常登录，且看到的菜单与权限矩阵一致
- [ ] 直接访问无权限页面的 URL 时，显示权限不足提示并重定向
- [ ] 角色切换后（由管理员变更），菜单和功能即时更新
- [ ] 所有关键操作均记录审计日志
