# 菜单结构与导航

## 1. 功能概述

平台采用层级化的侧边栏菜单导航，支持一级菜单和二级菜单，根据用户角色动态过滤可见菜单项，确保不同角色看到与其权限匹配的功能入口。

## 2. 菜单结构

```
┌─────────────────────────────┐
│  OSL Pay Logo               │  ← 黑色背景, 64px 高度
├─────────────────────────────┤
│  📊 快速开始                 │  ← 所有角色
│                              │
│  📈 仪表盘                   │  ← 管理员、业务人员
│                              │
│  📋 业务管理           [▼]   │  ← 一级菜单（可展开）
│    ├── 入驻申请              │  ← 管理员、业务人员
│    └── 成员与权限            │  ← 仅管理员
│                              │
│  🛠 开发者套件         [▼]   │  ← 一级菜单（可展开）
│    ├── API 文档              │  ← 管理员、技术人员
│    ├── API 凭证              │  ← 管理员、技术人员
│    ├── 域名白名单            │  ← 管理员、技术人员
│    ├── Webhook 管理          │  ← 管理员、技术人员
│    ├── 签名工具              │  ← 管理员、技术人员
│    ├── API 请求日志          │  ← 管理员、技术人员
│    └── MCP 配置中心          │  ← 管理员、技术人员
└─────────────────────────────┘
```

## 3. 交互行为

### 3.1 菜单展开/收起
- 点击含子菜单的一级菜单项，展开或收起子菜单列表
- 展开时显示向下箭头，收起时显示向右箭头
- 当前路径匹配某个子菜单时，自动展开其父菜单

### 3.2 激活状态
- **激活菜单项**: 黑色背景 + 白色文字 + 荧光绿图标（`#c4ff0d`）
- **非激活菜单项**: 浅灰背景（`#f8f9fa`）+ 深灰文字 + 灰色图标

### 3.3 权限过滤
- 根据用户角色过滤菜单项，仅显示有权访问的菜单
- 如果一级菜单下所有子菜单均无权限，则该一级菜单也隐藏

## 4. 布局规格

| 元素 | 规格 |
|------|------|
| 侧边栏宽度 | 240px（`w-60`），固定定位 |
| Logo 区域 | 黑色背景，64px 高度 |
| 导航区域 | 白色背景，`px-4 py-6 space-y-2` |
| 菜单项间距 | `space-y-2`（8px） |
| 菜单项内边距 | `px-3 py-2` |
| 菜单项圆角 | `rounded-lg` |
| 顶部栏 | 黑色背景，64px 高度，含环境切换 + 用户信息 |
| 内容区域 | `flex-1 overflow-y-auto p-6` |

## 5. 响应式行为

| 断点 | 行为 |
|------|------|
| ≥ 1024px（lg） | 侧边栏常显 |
| < 1024px | 侧边栏隐藏，通过汉堡菜单按钮打开（抽屉式） |
