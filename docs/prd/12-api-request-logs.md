# API 请求日志

## 1. 功能概述

API 请求日志模块展示最新 10 条 API 请求流水记录，帮助技术人员快速排查集成错误。支持查看完整的 Request 和 Response 内容。

**适用角色**: 管理员、技术人员

## 2. 日志列表

### 2.1 列表字段

| 字段 | 说明 |
|------|------|
| 时间戳 | 精确到秒 |
| HTTP 方法 | GET / POST / PUT / DELETE |
| 请求路径 | API 端点路径 |
| 状态码 | HTTP 响应状态码 |
| 响应耗时 | 毫秒（ms） |

### 2.2 颜色标识

**HTTP 方法颜色**:
| 方法 | 标签颜色 |
|------|----------|
| GET | 绿色 |
| POST | 蓝色 |
| PUT | 黄色 |
| DELETE | 红色 |

**状态码颜色**:
| 状态码范围 | 颜色 | 含义 |
|------------|------|------|
| 2xx | 绿色 | 成功 |
| 4xx | 黄色 | 客户端错误 |
| 5xx | 红色 | 服务器错误 |

### 2.3 自动刷新
- 每 5 秒自动刷新
- 仅保留最新 10 条记录
- 新请求进入时，最旧的记录自动淘汰

## 3. 日志详情

点击日志记录或「查看详情」按钮，打开详情模态框：

### 3.1 请求信息
- HTTP 方法 + 请求路径
- 状态码 + 响应耗时

### 3.2 Request Body
- JSON 格式化显示（带语法高亮）

### 3.3 Response Body
- JSON 格式化显示（带语法高亮）

### 3.4 模态框操作
- 点击右上角「×」关闭
- 点击「关闭」按钮关闭
- 点击遮罩层关闭
- 按 Escape 键关闭

## 4. 使用场景

| 场景 | 操作方式 |
|------|----------|
| 排查 API 调用失败 | 找到 4xx/5xx 状态码的记录，查看 Response Body 中的错误信息 |
| 验证签名是否正确 | 找到对应请求，401 = 签名失败，200 = 签名成功 |
| 检查请求参数 | 查看 Request Body，确认参数完整性和格式 |

## 5. 限制说明

- 仅保留最新 10 条，无历史翻页
- 敏感信息（密钥、密码）不记录
- 不支持导出
- 生产/沙箱环境日志独立
- 日志用于调试，不作为审计依据
