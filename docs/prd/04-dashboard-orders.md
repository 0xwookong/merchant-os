# 仪表盘与订单管理

本文档覆盖两个独立页面：**仪表盘**（`/dashboard`）和**订单管理**（`/transactions/orders`）。

---

## Part A — 仪表盘

### A1. 功能概述

仪表盘是商户登录后的业务运营总览页面，通过关键指标、交易趋势和支付方式分布三个维度帮助商户快速掌握业务健康度。

**适用角色**: 管理员、业务人员

### A2. 页面布局

```
┌─────────────────────────────────────────────────────────────┐
│  时间范围: [今日] [近 7 天] [近 30 天] [自定义 📅]            │

│                                                              │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐       │
```

### A3. 关键指标卡片

顶部展示 4 个核心指标卡片，所有指标与时间范围联动：

| 指标 | 说明 | 展示格式 | 图标 |
|------|------|----------|------|
| 交易总额 | 选定时段内的累计交易金额 | $XXX,XXX.XX + 环比变化率 | BanknotesIcon |
| 成功率 | 交易成功百分比（SUCCESSED + COMPLETED / 总数） | XX.X% + 环比变化率 | CheckBadgeIcon |
| 订单数量 | 选定时段内的总订单数 | X,XXX + 环比变化率 | ShoppingCartIcon |
| 活跃用户数 | 选定时段内产生订单的独立终端用户数 | X,XXX + 环比变化率 | UsersIcon |

**环比变化率**：与前一个等长时段对比。例如「近 7 天」对比上一个 7 天。
- 上升：绿色 `↑` 箭头
- 下降：红色 `↓` 箭头
- 无变化：灰色 `—`

**交互**：
- Hover 时卡片升起（`shadow-md`）
- 卡片内数值使用 `text-2xl font-semibold`，环比率使用 `text-sm`

### A4. 时间范围筛选

| 选项 | 说明 | X 轴粒度 |
|------|------|----------|
| 今日 | 当天 00:00 至当前时间 | 每小时（0:00, 1:00, ..., 23:00） |
| 近 7 天 | 最近 7 个自然日 | 每天（Mon, Tue, ...） |
| 近 30 天 | 最近 30 个自然日 | 每天（03/01, 03/02, ...） |
| 自定义 | 用户选择起止日期（最多 90 天） | ≤7 天按小时，≤30 天按天，>30 天按周 |

**交互**：
- 前三个选项为 Tab 切换样式，选中项黑底白字
- 「自定义」点击后弹出日期范围选择器（两个日期输入框 + 确认按钮）
- 切换时间范围后，指标卡片、趋势图、分布图全部联动刷新
- 刷新期间显示骨架屏（`animate-pulse`）

### A5. 交易趋势图

使用面积图（Area Chart）展示交易额随时间的变化趋势，叠加折线展示订单数。

**数据维度**：
| 维度 | Y 轴 | 图形 | 颜色 |
|------|------|------|------|
| 交易额（法币） | 左 Y 轴 | 渐变面积 + 实线 | 主色（黑色/深灰 → 透明渐变填充） |
| 订单数 | 右 Y 轴 | 虚线 | `var(--gray-400)` |

**X 轴时间粒度**：随时间范围自动调整（见 A4 表格）。

**交互**：
- Hover 显示 Tooltip：时间点、交易额、订单数
- Tooltip 样式：白色卡片 + `shadow-lg` + `rounded-lg`
- 图表区域高度固定 320px
- 包裹在白色卡片中，标题 "交易趋势"

**空态**：选定时段无数据时，图表区域显示居中文案 "暂无交易数据"。

**技术方案**：使用 Recharts（React 轻量图表库），组件：`<AreaChart>` + `<Area>` + `<Line>` + `<XAxis>` + `<YAxis>` + `<Tooltip>`

### A6. 支付方式分布

左右分栏布局：左侧环形图（Donut Chart），右侧明细表格。

**环形图**：
- 展示各支付方式占总交易额的比例
- 内圈空心，中心显示总交易额
- 每个扇区颜色对应支付方式：
  - Card: `var(--gray-900)` (深灰/黑)
  - Google Pay: `var(--gray-400)` (中灰)
  - Apple Pay: `var(--gray-200)` (浅灰)
- Hover 扇区时高亮，显示具体金额和百分比

**明细表格**：
| 列 | 说明 |
|----|------|
| 颜色标识 | 与环形图扇区颜色一致的小圆点 |
| 支付方式 | Card / Google Pay / Apple Pay |
| 交易额 | $XXX,XXX.XX |
| 占比 | XX.X% |
| 订单数 | X,XXX |

**布局**：
- 包裹在白色卡片中，标题 "支付方式分布"
- 左右比例约 4:6（环形图 : 明细表）
- 移动端改为上下布局

**空态**：无数据时显示灰色环形 + "暂无数据"。

**技术方案**：Recharts `<PieChart>` + `<Pie>` (innerRadius/outerRadius 实现环形)

### A7. API 设计

#### A7.1 仪表盘指标

```
GET /api/v1/dashboard/metrics?range=7d
GET /api/v1/dashboard/metrics?startDate=2026-03-01&endDate=2026-03-26
```

**响应**：
```json
{
  "code": 0,
  "data": {
    "range": "7d",
    "metrics": [
      {
        "key": "totalAmount",
        "label": "交易总额",
        "value": "128450.00",
        "unit": "USD",
        "changeRate": 12.3
      },
      { "key": "successRate", "label": "成功率", "value": "96.8", "unit": "%", "changeRate": 0.5 },
      { "key": "orderCount", "label": "订单数量", "value": "1234", "unit": "", "changeRate": 8.2 },
      { "key": "activeUsers", "label": "活跃用户数", "value": "856", "unit": "", "changeRate": 15.1 }
    ]
  }
}
```

#### A7.2 交易趋势（新增）

```
GET /api/v1/dashboard/trend?range=7d
GET /api/v1/dashboard/trend?startDate=2026-03-01&endDate=2026-03-26
```

**响应**：
```json
{
  "code": 0,
  "data": {
    "range": "7d",
    "granularity": "day",
    "points": [
      { "time": "2026-03-20", "amount": 18200.50, "orderCount": 176 },
      { "time": "2026-03-21", "amount": 21050.00, "orderCount": 198 },
      { "time": "2026-03-22", "amount": 19300.75, "orderCount": 182 }
    ]
  }
}
```

`granularity` 取值：`hour`（今日/≤7 天自定义）、`day`（7d/30d/≤30 天自定义）、`week`（>30 天自定义）。

```
GET /api/v1/dashboard/payment-methods?startDate=2026-03-01&endDate=2026-03-26
```

**响应**：

```json
  "data": {
    "range": "7d",
    "methods": [
      { "method": "CARD", "label": "Card", "amount": 83492.50, "orderCount": 802, "percentage": 65.0 },
```

## Part B — 订单管理

> 页面路径：`/transactions/orders`
> 适用角色：管理员、业务人员

### B1. 功能概述

订单管理页面提供完整的订单查询、筛选、详情查看和数据导出能力，是商户日常运营和客服处理的核心工具。

### B2. 订单列表

#### B2.1 列表字段

| 列名 | 说明 |
|------|------|
| 订单号 | 唯一标识，如 ORD20240303001，等宽字体 |
| 金额 | 法币金额 + 币种（如 $100.00 USD） |
| 加密货币 | 目标加密货币及数量 + 网络（如 0.05 ETH (Ethereum)） |
| 支付方式 | CARD / GOOGLEPAY / APPLE PAY，Badge 样式 |
| 状态 | 见 B5 状态说明，彩色 Badge |
| 创建时间 | 本地化日期时间格式 |

#### B2.2 筛选条件

| 筛选项 | 类型 | 选项 |
|--------|------|------|
| 订单号搜索 | 文本输入 | 支持模糊搜索 |
| 状态 | 下拉选择 | 全部 / CREATED / PROCESSING / SUCCESSED / COMPLETED / FAILED |
| 支付方式 | 下拉选择 | 全部 / CARD / GOOGLEPAY / APPLE PAY |
| 时间范围 | 日期选择器 | 开始日期 — 结束日期 |

#### B2.3 分页

- 每页显示 20 条记录（默认）
- 提供「上一页」「下一页」翻页按钮
- 显示当前页码和总记录数

### B3. 订单详情

点击订单行打开详情弹窗（Dialog），包含：

#### B3.1 基本信息
- 订单号、状态（Badge）、支付方式、创建时间
- 法币信息（金额、币种）
- 加密货币信息（金额、币种、目标网络、钱包地址）

#### B3.2 状态流转时间线

```
CREATED → PROCESSING → SUCCESSED → COMPLETED
                    └→ FAILED
```

时间线组件展示每个状态节点的变更时间，当前状态高亮。

#### B3.3 链上信息

| 字段 | 说明 |
|------|------|
| 交易哈希（TX Hash） | 可点击跳转到对应链的区块浏览器 |
| 链上确认数 | 当前确认数 |
| 区块高度 | 所在区块编号 |

仅在 SUCCESSED / COMPLETED 状态下展示链上信息区域。

### B4. 数据导出

- 点击「导出 CSV」按钮
- 根据当前筛选条件（状态、支付方式、时间范围）导出
- CSV 文件包含所有订单列表字段
- UTF-8 BOM 编码，支持 Excel 直接打开

### B5. 订单状态说明

| 状态 | 中文 | 含义 | 标签颜色 |
|------|------|------|----------|
| CREATED | 已创建 | 订单已创建，等待支付 | 灰色（gray-100/gray-700） |
| PROCESSING | 处理中 | 支付处理中 | 蓝色（blue-50/blue-700） |
| SUCCESSED | 支付成功 | 法币支付成功，等待加密货币到账 | 黄色（yellow-50/yellow-700） |
| COMPLETED | 已完成 | 加密货币已到账，订单完成 | 绿色（green-50/green-700） |
| FAILED | 失败 | 支付失败或订单异常 | 红色（red-50/red-700） |

### B6. 订单管理验收标准

- [ ] 订单列表正确加载并展示所有字段
- [ ] 订单号搜索支持模糊匹配
- [ ] 状态、支付方式筛选功能正常
- [ ] 时间范围筛选功能正常
- [ ] 点击订单行打开详情弹窗，展示完整信息
- [ ] 状态流转时间线展示正确
- [ ] TX Hash 可点击跳转区块浏览器
- [ ] CSV 导出按当前筛选条件正确导出
- [ ] 分页功能正常（每页 20 条）
- [ ] Loading / Empty / Error 状态均有对应展示
